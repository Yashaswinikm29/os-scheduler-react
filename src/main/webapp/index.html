Key Formulas Reference
Turnaround Time (TAT)
TAT = Completion Time − Arrival Time
Total time from process arrival to completion. Includes both waiting and execution time.
Waiting Time (WT)
WT = Turnaround Time − Burst Time
Time the process spends waiting in the ready queue before getting CPU.
Average Waiting Time
AWT = Σ(Waiting Time) ÷ n
Sum of all waiting times divided by total number of processes. Lower is better.
Average Turnaround Time
ATAT = Σ(Turnaround Time) ÷ n
Sum of all turnaround times divided by total number of processes. Lower is better.
CPU Utilization
CPU% = (Busy Time ÷ Total Time) × 100
Percentage of time the CPU is actively executing processes vs sitting idle.
Throughput
Throughput = n ÷ Total Time
Number of processes completed per unit time. Higher throughput = better performance.
Response Time
RT = First CPU Time − Arrival Time
Time from arrival to first CPU access. Critical for interactive systems.
Round Robin Cycles
Cycles = ⌈Burst Time ÷ Quantum⌉
Number of times a process is scheduled before completion in Round Robin.

FCFS
First Come First Serve · Non-Preemptive
Processes execute in arrival order. Simple queue — first in, first out. No priority or burst time considered.
Simple to implement No starvation Convoy effect High avg wait
Best for: Batch jobs, print queues
Avoid when: Short processes behind long ones
CPU overhead: Very Low
SJF
Shortest Job First · Non-Preemptive
Process with shortest burst time runs first. Provably optimal for average waiting time among non-preemptive algorithms.
Optimal AWT High throughput Starvation risk Needs burst time
Best for: Batch systems with known burst times
Avoid when: Burst times unknown in advance
CPU overhead: Low
Priority
Priority Scheduling · Non-Preemptive
Each process has a priority number. Lower number = higher priority. Critical tasks always get CPU first.
Critical tasks first Flexible control Starvation Priority assignment needed
Best for: Real-time systems, OS kernels
Avoid when: All processes equally important
CPU overhead: Medium
Round Robin
Round Robin · Preemptive
Each process gets equal CPU time (time quantum). Preempts after quantum expires and goes to back of queue.
Fair to all processes No starvation Best response time Context switch overhead
Best for: Interactive & time-sharing systems
Avoid when: Quantum too small (high overhead)
CPU overhead: Medium–High
